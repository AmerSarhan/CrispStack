# Contributing to CrispStack

Thanks for your interest in contributing! Here's how to get started.

## Development Setup

```bash
git clone https://github.com/AmerSarhan/CrispStack.git
cd CrispStack
npm install
cp .env.example .env.local
npm run dev
```

## Pull Requests

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Run `npm run build` to ensure the build passes
4. Run `npm run lint` to check for lint errors
5. Submit a pull request

## Guidelines

- Keep changes focused and minimal
- Follow existing code patterns and naming conventions
- Test both light and dark themes
- Test responsive layouts (mobile, tablet, desktop)
- No unnecessary dependencies

## Reporting Bugs

Use the [GitHub issue tracker](https://github.com/AmerSarhan/CrispStack/issues) with the bug report template.

## Feature Requests

Open an issue with the feature request template. We prioritize features that align with CrispStack's philosophy: **quality over quantity**.
